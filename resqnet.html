How to use the Simulator — Step by step

This guide explains every input and how to run the simulation. Follow the steps below in order.

  • Step 1 — Choose Sender: From the Sender dropdown pick the node which should originate the message (e.g., Node A). This is the device that sends the encrypted packet.
  • Step 2 — Choose Receiver: From the Receiver dropdown pick a target node or choose BROADCAST to send to all nodes.
  • Step 3 — Set Passphrase: The Passphrase is the shared symmetric key used for encryption in this demo. Keep it the same on sender & receiver to simulate a trusted session. Change it to simulate different sessions or failed decryption.
  • Step 4 — Type Message: Enter the plaintext message (e.g., HELP or LOCATION). Short messages work best for demo clarity.
  • Step 5 — Send: Click Send Secure Message. The UI will show the base64 encrypted payload and display the decrypted text at the receiver (if passphrase matches).
  • Step 6 — Read Logs: The Message Log records the timestamp, sender, receiver, encrypted payload and decrypted result. Use Download Log (CSV) to export or Clear Log to reset.
  • Step 7 — Path Table: When a packet traverses intermediate nodes, its actual path (e.g., A → E → F → I) is logged in the Path Followed table. Use Download Path Log (CSV) to export it.
  • Notes on BROADCAST: If you pick BROADCAST, the simulator sends the encrypted message to every node. Each node will attempt to decrypt using the passphrase you provided.

Troubleshooting & Tips

  • Wrong decrypted output: Make sure both sender and receiver use the same passphrase. Change passphrase to test decryption failure.
  • No message sent: Ensure you selected a Sender and a Receiver and entered a message.
  • Log missing path entries: Some runs (e.g., direct neighbors) have minimal hops; intermediate nodes are only shown when routing is simulated.
  • Export issues: CSV downloads open in spreadsheet apps; use "Save As" when browser shows the CSV content.

Shortcuts & tips:

On mobile, use portrait view and scroll — controls collapse under the network. The passphrase is visible for educational purposes only — never expose real keys in production.

Learn — Network Concepts Used in ResQNet

Demo Features
  • Encrypt & decrypt messages (base64 display)
  • Broadcast vs unicast simulation
  • Message & path logs with CSV export
Learning Outcomes
  • How symmetric passphrases affect decryption
  • How mesh nodes forward packets
  • How logs and packet paths help debugging
Dijkstra’s Algorithm (Shortest Path Routing)
  • Calculates the shortest path from a source to all other nodes in a weighted graph.
  • Used in real-world routing protocols like OSPF to create efficient, loop-free routes.
  • Ensures reliable and fast delivery of packets in large mesh networks.
Flooding Technique (Broadcast Routing)
  • Every incoming packet is sent out through all output links except the one it arrived on.
  • Guarantees that the message reaches every node, even without a predefined route.
  • Ideal for emergency or highly dynamic networks, but can cause traffic overload.
Mesh-based D2D Communication
  • Nodes act as both end devices and relays, forwarding data for others.
  • Creates a self-healing and adaptive network — no single point of failure.
  • Used for communication in disaster zones when cell towers are inaccessible.
AES Encryption (Secure Communication)
  • Advanced Encryption Standard, widely used in modern cybersecurity applications.
  • Symmetric encryption: same key used for encrypting and decrypting messages.
  • Ensures privacy, integrity, and security even when passed through multiple relays.
Packet Logs & Route Tracing
  • Logs the entire path followed by each packet through the network.
  • Helps analyze message flow and routing behavior.
  • CSV export helps offline analysis and report preparation.
Networking in Disaster Relief Scenarios
  • No dependence on centralized wireless or tower infrastructure.
  • Allows communication between first responders and affected civilians.
  • Built for remote or disrupted areas with minimal power.
