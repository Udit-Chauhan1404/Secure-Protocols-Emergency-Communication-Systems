Designing a Secure Communication Protocol for Disaster Relief Networks

A ResQNet Project | BCSE308L - Computer Networks

Area & Domain of Research

Area: Computer Networks

Domain: Secure Protocols & Emergency Communication Systems

Objective

The aim of this project is to develop a secure, lightweight, and reliable communication protocol that can function during natural disasters when regular telecom infrastructure is unavailable. The protocol will work over a mesh network, allowing encrypted device-to-device (D2D) communication among rescue teams. The project also includes a digital storytelling component to visualize the disaster situation and the effectiveness of our solution.

Explanatory Video

Background / Short Description

Natural disasters often damage mobile towers, power lines, and internet services, causing a total breakdown of communication. In such cases, emergency responders need a secure, decentralized method to coordinate rescue operations. Traditional methods like walkie-talkies lack encryption and scalability. Our proposed solution aims to fill this gap using principles of computer networks and lightweight encryption.

Motivation

Why is this important?

Communication is critical in emergencies; delays can cost lives.

Who benefits?

Emergency responders, disaster management teams, affected civilians.

Impact:

Improves disaster readiness, supports SDG 9 (resilient infrastructure), and lays the foundation for scalable emergency systems.

Problem Statement

How can we ensure secure and reliable communication between disaster relief workers when the internet and telecom networks are unavailable?

Prevailing Techniques & Our Innovation

Existing Solutions:

  • Walkie-talkies (not encrypted)
  • Satellite phones (expensive, limited access)
  • HAM radio (requires licensing and setup time)

Our Innovation:

  • Cost-effective, encrypted communication protocol
  • Uses existing mobile hardware or IoT nodes
  • Mesh-based network — no single point of failure

Working Principle

The secure communication protocol operates on a mesh-based network designed for disaster areas with no existing connectivity. The core components involved are:

  • Node Initialization: Each device joins the mesh network and generates a unique device identifier.
  • Key Generation: Lightweight symmetric encryption (e.g., AES-128) is used. Devices exchange keys using a secure handshake.
  • Message Encryption: The sender encrypts the rescue message using the shared session key.
  • Multi-hop Forwarding: The encrypted packet is forwarded across nearby nodes using the mesh routing protocol.
  • Decryption at Destination: Only the intended node with the correct key can decrypt the message.

This protocol ensures confidentiality, authenticity, and resilience even when traditional telecom services fail.

Expected Inputs / Outputs / Results

Inputs: Device identifiers, encryption key, message content

Outputs: Encrypted message sent through mesh; decrypted at destination

Result: Successful transmission of rescue messages over a disaster-resilient secure protocol

Outcome

A conceptual secure disaster communication protocol that enables encrypted D2D communication in disconnected environments, improving coordination and safety during disasters.
